## Архитектура проекта

Проект полностью отвязан от Firebase и переведён на связку **GitHub Pages + Google Sheets + Supabase**.

### 1. Как работают курсы (shatkarm.ru/course, /fire, /library)
- **Файлы курса:** Логика и контент курсов хранятся прямо в файле `index.html` каждой папки (например, `/course/index.html` или `/fire/index.html`).
- **База учеников:** Список доступов (коды, имена, даты старта) теперь хранится прямо внутри HTML в объекте `STUDENTS`.
- **Защита от складчины:** Мы используем Supabase (`device_bindings`) для жесткой привязки 1 код = 1 устройство. Если ученик передаст код кому-то другому, система заблокирует вход на новом устройстве.

### 2. Как добавить ученика на курс
Вам больше не нужно лезть в сложные базы данных. Просто откройте файл нужного курса (например, `/course/index.html` для "Восстановление тела"):

1. Найдите объект `STUDENTS`:
```javascript
let STUDENTS = {
    'DEMO01': { name: 'Демо-ученик', start: '2026-04-15', course: 'recovery', days: 14 },
    'NEWCODE': { name: 'Имя Ученика', start: '2026-04-13', course: 'recovery', days: 14 }
};
```
2. Добавьте новую строчку с уникальным кодом и датой старта потока.
3. Сохраните изменения (Commit) — и ученик сразу сможет войти!

### 3. Как управлять базой устройств (Supabase)
Если ученик поменял телефон и просит сбросить привязку:
1. Зайдите в админ-панель: `shatkarm.ru/course/admin/`
2. Введите пароль: `admin2026`
3. Найдите код ученика и нажмите кнопку **«Сбросить устройство»**. Запись в Supabase будет удалена, и он сможет войти с нового телефона.

## Как редактировать контент курсов

Всё расписание и тексты хранятся в объекте `COURSES` внутри HTML-файла курса.

1. Откройте нужный файл (например, `/course/index.html`).
2. Найдите блок дней нужного курса. Пример:
```javascript
{ day: 4, phase: 2, title: 'Начало диеты', icon: '🧘', desc: 'Важные лекции❗️',
  content: `<h3>Полная лекция</h3>
  <p>Текст лекции...</p>
  <div class="video-container" style="...">
    <iframe src="ССЫЛКА_ВК_HD=3" ...></iframe>
  </div>` 
},
```
3. Вы можете свободно менять названия, текст, добавлять HTML-теги (`<strong>`, `<ul>`) и вставлять ссылки на видео из VK Video.
4. **Важно:** Все видео из VK должны иметь параметр `hd=3` в ссылке, чтобы они грузились быстро и в хорошем качестве.
5. Не забывайте запятые между днями!

## Строгие правила безопасности кода (⚠️ ВАЖНО)

Айса не программист. Любая ошибка в синтаксисе (забытая запятая, незакрытая кавычка) сломает сайт для всех учеников. 

Поэтому для ИИ-ассистента действуют строгие правила:
1. **Никаких коммитов вслепую:** Перед каждым `git push` ассистент обязан сделать синтаксическую проверку локально (например, `node -c index.html` или через Python).
2. **Никаких Markdown-файлов для документов:** Все ТЗ, чек-листы и выгрузки кода должны сохраняться только в `.docx` или `.pdf`.

## Если сайт сломался (Белый экран)

1. Зайдите на GitHub → нужный файл `index.html` → **History** (вверху справа).
2. Найдите последний рабочий коммит (до поломки).
3. Нажмите `...` → **Revert changes**.
4. GitHub отменит последние изменения и вернёт рабочий код.
