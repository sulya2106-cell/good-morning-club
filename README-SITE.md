## 🔐 Как работает платформа курсов

### Коды доступа
Ученики получают **персональный код** при оплате. Код привязывается к устройству через **Supabase** — один код = одно устройство.

### Google Sheets
Данные учеников хранятся в Google Sheets. Таблица подключена к сайту через публичную CSV-ссылку.

### Supabase (привязка устройств)
- **URL:** `https://tbkfhxkovdwsfxvyouyu.supabase.co`
- При первом входе device ID записывается в базу
- Повторный вход с другого устройства — блокировка
- Если ученик сменил телефон: зайди в Supabase → Table Editor → `device_bindings` → удали строку с его кодом

### Три курса на платформе
1. **Восстановление тела** — 9 дней, контент заполнен из телеграм-групп
2. **Внутренний огонь** — 30 дней, 4 фазы
3. **Хатха-йога** — открытый формат

## ✏️ Как менять тексты

1. Открой нужный файл на GitHub
2. Найди текст, который хочешь поменять
3. Замени его (не трогай символы `<` и `>`)
4. Нажми **Commit changes**
5. Через 1-2 минуты изменения появятся на сайте

### Как менять цены
В файлах `recovery/index.html` и `fire/index.html` найди строки с ценами:
```html
<div class="price-val">7 000₽</div>
```
Замени цифру на нужную.

## 🎥 Как добавить видео в библиотеку

1. Открой `content.js`
2. Найди массив `PRACTICES`
3. Добавь новое видео по шаблону:

```javascript
{
    t: 'Название видео',          // название
    c: 'body',                     // категория: body, recovery, breath, nauli, theory
    d: '20 мин',                   // длительность
    lv: 'Все уровни',             // уровень
    desc: 'Описание...',          // описание
    vk: 'ССЫЛКА_ИЗ_ВК',          // ссылка для встраивания
    e: '🧘'                       // эмодзи
},
```

4. **Не забудь запятую** после `}` если после видео идёт ещё одно!

### Как получить ссылку на видео из ВК
1. Открой видео ВКонтакте **с компьютера**
2. Нажми **"Поделиться"** → вкладка **"Экспортировать"**
3. В поле "Код для вставки" найди `src="..."`
4. Скопируй только ссылку внутри кавычек (вида `https://vk.com/video_ext.php?oid=-123...`)

## 📝 Как добавить текстовую карточку в библиотеку

1. В `content.js` найди массив `TEXTS`
2. Добавь по шаблону:

```javascript
{
    t: 'Название',
    c: 'theory',
    e: '📖',
    sub: 'Короткое описание',
    h1: 'Заголовок',
    h2: 'Подзаголовок',
    body: '<p>Текст. Для абзацев — теги p.</p>'
},
```

## 📸 Как поменять фото

1. Загрузи фото в репозиторий на GitHub (формат `.jpg` или `.png`, сжатое)
2. В HTML файле найди тег `<img src="...">`
3. Замени путь в `src` на название нового файла

---

## 🚀 Как публиковать изменения

1. Измени файл на GitHub
2. Нажми зелёную кнопку **"Commit changes..."**
3. Подтверди — **"Commit changes"**
4. Подожди 1-2 минуты — GitHub Pages обновит сайт
5. Если на сайте старая версия — обнови с очисткой кэша: `Ctrl + F5` (Windows) или `Cmd + Shift + R` (Mac)

## 🛑 Что нельзя трогать

- В `course/index.html` — блоки JavaScript внизу (логика авторизации и Supabase)
- Файлы `robots.txt` и `sitemap.xml` — для SEO, не трогай
- Не удаляй HTML-теги (`<div>`, `</section>` и т.д.)
- В `content.js` — следи за кавычками `' '` и запятыми `,`

## 🩹 Частые проблемы

| Проблема | Решение |
|----------|---------|
| Библиотека — белый экран | В `content.js` пропущена запятая или кавычка |
| Текст поехал | Случайно удалён HTML-тег |
| Видео не играет | Проверь ссылку `vk` — нужна ссылка для экспорта (`video_ext.php`), не обычная |
| Сайт не обновился | Очисти кэш: `Ctrl + F5` |
| Ученик не может войти | Проверь код в Google Sheets и строку в Supabase → `device_bindings` |
| Ученик сменил телефон | Удали его строку в Supabase, он привяжется заново |
